# 年度更新タブ 列拡張・表示切替・列固定 設計書（サブプロジェクト①）

**作成日：** 2026-07-19
**ステータス：** 承認済み
**関連ドキュメント：** `docs/superpowers/specs/2026-07-19-renewal-tab-branch-quick-toggle-design.md`（枝番別クイック提出済切替、本機能の前提）

---

## 1. 背景・全体スコープ

年度更新タブの一覧に、名簿タブ（`member_tab.py`）と同等の列・列カスタマイズ機能（表示/非表示・列固定・集約並び替え・チェックボックス選択によるラベル出力/メール送信）を追加してほしいという依頼を受けた。機能規模が大きいため、3段階のサブプロジェクトに分割して進める。

1. **①列拡張＋表示/非表示＋列固定**（本設計書の対象）
2. ②集約並び替えボタン
3. ③チェックボックス選択列＋ラベル出力・メール送信ボタン

---

## 2. 実装テンプレート

`withdrawn_tab.py`（委託解除済みタブ）が、名簿タブと同じ列カスタマイズ機能を独自の追加列（委託解除日・委託解除理由）と組み合わせて実装した直近の前例であり、これを実装テンプレートとする。`member_tab.py`が公開する共有クラスをそのままインポートして再利用する：

```python
from app.ui.member_tab import (
    SortableTableWidgetItem, _SelectionDelegate, _CheckHeader,
    _FrozenCheckDelegate, _FrozenItemDelegate,
)
```

固定列オーバーレイ（`QTableView`同期）・列ヘッダ右クリックメニュー・表示列選択ダイアログ・列幅/順序/表示状態の永続化ロジックは、`withdrawn_tab.py`の該当メソッド（`_build_ui`の固定ビュー構築部分、`_update_frozen_view_geometry`、`_on_frozen_header_clicked`、`_on_frozen_clicked`、`_on_frozen_double_clicked`、`_on_frozen_context_menu`、`_set_freeze_col`、`_exec_column_menu`相当、`_on_header_context_menu`相当、列幅/順序保存のシグナルハンドラ）を年度更新タブ用に移植する。

---

## 3. 列構成

チェックボックス列は本サブプロジェクトでは追加しない（③で追加）。

```
管理No., 会, 会員No., 事業所名, フリガナ, 所属・役職, 代表者名, 代表者フリガナ, メール,
市外局番, 電話番号, FAX市外局番, FAX, 郵便番号, 住所,
郵送先郵便番号, 郵送先住所, 郵送先宛名, 雇用保険事業所番号,
枝番0, 枝番2, 枝番4, 枝番5, 枝番6,
特別, 継続一括, 登録日, 最終更新日,
最終対応日（全体）, メモ（全体）,
全体状況, 最終対応日（年度更新）, 備考（年度更新）
```

- 名簿タブのCOLS順序をそのまま踏襲し、名簿タブの「0/2/4/5/6」（保険番号単体列）の位置に、既存の枝番セル（保険番号＋提出状況を1セルに表示、クリックでトグル）をそのまま差し込む。
- 「特別」「継続一括」は名簿タブと同じロジック（`any(e.is_tokubetsu for e in m.insurance_entries)` / `is_ikkatsu`）で算出する。
- 「最終対応日（全体）」「メモ（全体）」は`Member`単位の全体対応履歴・メモ（名簿タブの「最終対応日」「メモ」と同じデータソース）。
- 「全体状況」「最終対応日（年度更新）」「備考（年度更新）」は既存の年度更新レコード固有列（`AnnualRenewal.overall_status` / `last_contacted_at` / `note`）で、末尾に維持する。
- 列ラベルの重複（「最終対応日」）を避けるため、全体対応履歴側は「最終対応日（全体）」「メモ（全体）」、年度更新固有側は「最終対応日（年度更新）」「備考（年度更新）」と明示的に区別する。

## 4. データ取得

- `RenewalService.search()`は既に`member`の全スカラー列と`member.insurance_entries`を事前ロード済み（前回の枝番クイック切替機能で対応済み）のため、追加のサービス層変更は不要。
- 「最終対応日（全体）」「最終更新日」は`ActivityService`（`app/services/activity_service.py`、名簿タブが使用しているものと同じサービス）の`get_last_logged_at_map(member_ids)` / `get_last_changed_at_map(member_ids)`をrenewal_tab側でも呼び出し、`RenewalTab._refresh()`内で`member_ids = [r.member.id for r in records]`から算出する。

---

## 5. 永続化キー

名簿タブ・委託解除済タブと衝突しない専用キー名を`staff_settings`に使う：

`renewal_hidden_columns` / `renewal_column_order` / `renewal_column_widths` / `renewal_freeze_col` / `renewal_sort_column` / `renewal_sort_order`

`_get_staff_setting` / `_set_staff_setting`ヘルパーは、`member_tab.py` / `withdrawn_tab.py`と全く同じ実装（`self._config.staff_settings[staff_name][key]`を読み書きし、`config_path`があれば保存）をrenewal_tab.pyにも追加する（既存2タブでも共通化されておらず個別実装のため、既存パターンを踏襲）。

---

## 6. 列固定（フリーズ）

名簿タブ・委託解除済タブと全く同じ挙動。

- 初期状態は固定なし（`freeze_col = 0`）。
- 列ヘッダを右クリックすると「ここまで固定」メニューが出て、選択した列までが左側に固定される（想定用途：事業所名列まで固定してスクロールしても管理No.・事業所名が常に見える状態にする）。
- 固定を解除する「列固定を解除」メニューも用意する。

---

## 7. 枝番セルのクリックトグルとの整合

既存の`_on_cell_clicked`は列インデックスを`BRANCH_COL_START`からの固定オフセットで判定していたが、列が大幅に増え、かつユーザーが列の表示/非表示・並び替え・固定を行えるようになるため、列インデックスの絶対値に依存せず、`COLS`内での枝番列の位置（`INS_TYPES`に対応する列だけを判定する集合）から動的に判定するよう変更する。列の非表示・並び替えを行っても、枝番セルのクリックトグルは引き続き正しい枝番に対して動作する。

---

## 8. UI

画面サイズは対象環境（1366×768、ウィンドウ幅1280px以内）を維持する。列数が大幅に増えるため、水平スクロール（固定列＋残りの列を横スクロール）は必須となる。これは名簿タブ・委託解除済タブと同じ前提であり、新たな制約ではない。

---

## 9. テスト計画

このタスクはUIのみの変更（サービス層への変更なし）のため、既存プロジェクトの慣例（`member_tab.py` / `withdrawn_tab.py` / `renewal_tab.py`のいずれも自動テストなし）に合わせ、自動テストは追加しない。手動確認（`python main.py`起動、または既存パターンに倣ったプログラムからのウィジェット検証スクリプト）で以下を確認する：

1. 一覧に全列が名簿タブと同じ順序・ラベルで表示される（枝番部分は既存の結合セル、末尾に年度更新固有3列）
2. 表示列選択ダイアログで列の表示/非表示を切り替えられ、職員別に保存・復元される
3. 列ヘッダ右クリックで「ここまで固定」→事業所名列まで固定できる。固定列は横スクロールしても常に表示される
4. 列幅変更・列並び替え（ドラッグ）が職員別に保存・復元される
5. 枝番セルのクリックトグル（未提出⇔提出済）が、列の表示/非表示・並び替え後も正しい枝番に対して動作する
6. 「特別」「継続一括」「最終対応日（全体）」「メモ（全体）」「最終更新日」が名簿タブと同じ値を表示する
