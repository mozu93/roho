# 年度更新タブ チェックボックス選択列・ラベル出力・メール送信 設計書（サブプロジェクト③）

**作成日：** 2026-07-19
**ステータス：** 承認済み
**関連ドキュメント：** `docs/superpowers/specs/2026-07-19-renewal-tab-column-expansion-design.md`（サブプロジェクト①）、`docs/superpowers/specs/2026-07-19-renewal-tab-aggregate-sort-design.md`（サブプロジェクト②）

---

## 1. 背景

年度更新タブの一覧に、名簿タブ（`member_tab.py`）・委託解除済タブ（`withdrawn_tab.py`）と同じチェックボックス選択機能（全選択ヘッダー・シフトクリック範囲選択）と、選択した事業所へのラベル出力・メール送信機能を追加する。これで3段階に分けたサブプロジェクトの最終段階となる。

クイック選択フィルタ（名簿タブの「特別加入のみ選択」「郵送先ありのみ選択」に相当するもの）は追加しない。既存の「全体状況」フィルタで一覧を絞り込んでから全選択ヘッダーを使えば同じ目的を達成できるため、対象外とする。

---

## 2. 実装テンプレート

`withdrawn_tab.py`のチェックボックス選択・シフトクリック範囲選択・ラベル出力・メール送信部分をそのまま踏襲する。`member_tab.py`の共有クラスを追加でインポートする：

```python
from app.ui.member_tab import (
    SortableTableWidgetItem, _SelectionDelegate, _CheckHeader, _FrozenCheckDelegate,
)
```

`LabelDialog(engine, members: list, parent=None)`・`ComposeEmailDialog(engine, config, members: list, parent=None)`はどちらも既存のダイアログをそのまま呼び出す（新規実装なし）。どちらも`Member`オブジェクトのプレーンなリストを受け取るため、チェックされた行から`r.member`を集めるだけでよい。

---

## 3. 列構成の変更

チェックボックス列を先頭（列0）に追加し、既存の33列は全て1つずつ後ろにずれる：

```
[チェックボックス],
管理No., 会, 会員No., 事業所名, フリガナ, 所属・役職, ... （以下サブプロジェクト①と同じ）
```

- `BRANCH_COL_START`：19 → 20
- `_TAIL_START`：24 → 25
- 「管理No.」列（新しい列1）は引き続き`r.id`（`AnnualRenewal.id`）をUserRoleに保持し、`_on_row_double_clicked`・`_on_cell_clicked`から参照する（変更なし、列番号がずれるだけ）
- 新しいチェックボックス列（列0）のモデルアイテムには`m.id`（`Member.id`）をUserRoleに保持する（選択状態の管理用）

---

## 4. 選択状態の管理

- `self._checked_ids: set[int] = set()`：選択中の`Member.id`の集合。検索・フィルタで一覧を絞り込んでも保持される（絞り込みで一時的に見えなくなった選択も、フィルタを戻せば復元される）
- `self._last_checked_member_id: int = -1`：シフトクリック範囲選択の起点
- `self._member_row_map: dict[int, int] = {}`：`Member.id` → 現在の行番号。`_fill_table`実行後・ソート変更後に再構築する
- チェックボックス自体は`QCheckBox`を`QWidget`コンテナに包んで`setCellWidget`で配置する（既存2タブと同じパターン）。選択状態の同期用にモデルアイテム（`setItem`）にも`Member.id`をUserRoleとして格納する

---

## 5. UI

### 5.1 ヘッダーチェックボックス（全選択）

`_CheckHeader`（`member_tab.py`）を`self._table.setHorizontalHeader()`に設定し、列0クリックで全選択/全解除をトグルする。**既存のヘッダー設定（ソート・列幅・列並び替え・右クリックメニューの接続）は、`_CheckHeader`に差し替えた後に行う**（`withdrawn_tab.py`と同じ順序。ヘッダーオブジェクトを差し替える前に接続すると失われるため）。

### 5.2 シフトクリック範囲選択

`_on_check_changed(member_id, state)`：Shift修飾キーが押されており、かつ直前にチェックした行が存在する場合、その行から今回チェックした行までの範囲を一括で同じ状態にする。`withdrawn_tab.py`の`_on_check_changed`をそのまま踏襲する。

### 5.3 列固定オーバーレイとの連携

- `self._frozen_view.setItemDelegateForColumn(0, _FrozenCheckDelegate(self._checked_ids, self._frozen_view))`
- 固定ビューのクリックハンドラ（`_on_frozen_clicked`、新規追加）：列0がクリックされた場合はチェック状態をトグルし、それ以外の列は行選択のみ行う（`withdrawn_tab.py`の`_on_frozen_clicked`と同じ）

### 5.4 表示列選択ダイアログ

チェックボックス列（列0）は表示/非表示の対象から除外する（常に表示）。`_exec_column_menu`のチェックボックス生成ループで列0をスキップする。

### 5.5 列固定メニュー

チェックボックス列（列0）は「ここまで固定」の対象から除外する（`freeze_col <= 0`が「固定なし」を表す番兵のため。サブプロジェクト①で管理No.列を除外していたのと同じ理由で、今回はチェックボックス列を除外する）。既存の条件式`0 < logical < len(COLS)`は変更不要（列0が管理No.からチェックボックスに変わるだけで、除外対象が列0である点は同じ）。

### 5.6 列幅

チェックボックス列は固定幅44pxとする（`member_tab.py`/`withdrawn_tab.py`と同じ）。列幅復元ループで列0を特別扱いする。

### 5.7 ボタン

一覧下部に新しいボタン行を追加し、「ラベル出力」「メール送信」ボタンを配置する。

- 「ラベル出力」：`self._records`のうちチェックされている`Member`を集めて`LabelDialog(self._engine, members, parent=self).exec()`を呼ぶ。未選択なら警告ダイアログ「出力する事業所を選択してください（左端のチェックボックスで選択）。」
- 「メール送信」：同様に`ComposeEmailDialog(self._engine, self._config, members, parent=self).exec()`を呼ぶ。未選択なら警告ダイアログ「送信先を選択してください（左端のチェックボックスで選択）。」

---

## 6. 既存機能との整合

- サブプロジェクト①・②で実装した機能（表示/非表示・列幅・列並び替え・列固定・ソート・集約並び替え・枝番セルのクリックトグル）は、列インデックスがすべて+1ずれる以外は変更しない
- `_fill_table`実行後、チェックボックスの状態を`self._checked_ids`から復元し、`self._member_row_map`を再構築する

---

## 7. テスト計画

UIのみの変更（サービス層への変更なし）のため、既存プロジェクトの慣例に合わせ自動テストは追加せず手動確認する：

1. チェックボックス列が列0に表示され、既存の32列（旧33列から表示切替対象は変わらず）は1つずつ後ろにずれて正しく表示される
2. ヘッダーのチェックボックスをクリックすると全選択/全解除される
3. Shiftキーを押しながら別の行をチェックすると、範囲選択される
4. 検索・フィルタで一覧を絞り込んでも、選択状態は保持される（絞り込みを解除すると選択済みのまま表示される）
5. チェックボックス列を「表示列選択」で非表示にできない（一覧に出てこない）
6. チェックボックス列を「ここまで固定」の対象にできない
7. 事業所を選択して「ラベル出力」→ 既存のラベル出力ダイアログが選択した事業所で開く
8. 事業所を選択して「メール送信」→ 既存のメール送信ダイアログが選択した事業所で開く
9. 未選択で「ラベル出力」「メール送信」を押すと警告が出る
10. 既存の枝番セルのクリックトグル・行ダブルクリックでの編集ダイアログ・列固定・ソート・集約並び替えが、列インデックスの変更後も正しく動作する

---

## 8. 今回除外する範囲

- クイック選択フィルタ（特別加入のみ・未提出のみ等）
- 一括提出済み変更などの新規一括操作
